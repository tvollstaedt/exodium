# Exodian

A cross-platform game launcher for the [eXoDOS](https://www.retro-exo.com/exodos.html) collection. Replaces the Windows-only LaunchBox frontend with a native desktop app that runs on Linux, macOS, and Windows.

## Features

- Browse 9,198 games across 4 collections (eXoDOS, German, Spanish, Polish)
- Stream individual games from the eXoDOS torrents on demand (no full 500 GB download needed)
- Launch games via DOSBox Staging with auto-patched configs
- Collection tabs for browsing each language pack separately
- Shortcode-keyed thumbnails for box art
- Save game backup/restore on uninstall
- Favorites: star any game to bookmark it for later download or play

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Shell | Tauri v2 |
| Frontend | SolidJS + TypeScript + Vite |
| UI Components | Ark UI (`@ark-ui/solid`) |
| Backend | Rust |
| Database | SQLite (pre-built, ships with the app) |
| Torrent | librqbit (selective file downloads) |

## Development

### Prerequisites

- [pnpm](https://pnpm.io)
- [Rust toolchain](https://rustup.rs) (bootstrapped automatically via `pnpm tauri dev`)
- [aria2](https://aria2.github.io) — required for `init-dev` only (`brew install aria2` / `apt install aria2`)
- Python 3 + [Pillow](https://python-pillow.org) — required for `init-dev` only (`pip3 install Pillow`)

> DOSBox Staging is bundled automatically by `init-dev` — no manual installation needed.

### First-time setup

```bash
pnpm install

# Download XODOSMetadata.zip via torrent and generate thumbnails (~2–5 GB, one-time)
pnpm run init-dev

pnpm tauri dev
```

`init-dev` is idempotent — safe to re-run; already-downloaded files and existing thumbnails are skipped. Use `--force` to regenerate thumbnails from scratch. The downloaded `XODOSMetadata.zip` is cached at `~/.exodian-dev/` (override with `XDO_DEV_DATA=/your/path`).

### Subsequent runs

```bash
pnpm tauri dev
```

### Useful scripts

| Command | Description |
|---------|-------------|
| `pnpm run init-dev` | First-time setup: download DOSBox binary + generate thumbnails |
| `pnpm run get-dosbox` | Download DOSBox Staging binary only (no thumbnails) |
| `pnpm run infohash` | Print stored torrent infohashes from the local DB |
| `pnpm tauri dev` | Start the app in development mode |

## Thumbnail Generation

`scripts/gen_thumbnails.py` resizes box art from a metadata ZIP into shortcode-keyed JPEGs. It now accepts two extra flags:

| Flag | Description |
|------|-------------|
| `--db <path>` | Path to `metadata/exodian.db` — highest-priority shortcode source; ensures LP-exclusive titles (e.g. Spanish/Polish games with no eXoDOS counterpart) get the correct filename |
| `--extra-xml <xml.gz>` | Additional catalogue to merge as a fallback (e.g. pass `MS-DOS.xml.gz` when generating GLP/SLP/PLP thumbs to resolve box art that belongs to EN-catalog games) |

`init-dev.sh` already passes both flags for all four collections — no manual intervention needed.

## Regenerating the Game Database

The pre-built SQLite database (`metadata/exodian.db.gz`) ships with the app and contains all game metadata, torrent indices, and shortcode mappings. To regenerate it from the source XML files:

```bash
cd src-tauri
cargo run --bin generate_db
gzip -k ../metadata/exodian.db
```
